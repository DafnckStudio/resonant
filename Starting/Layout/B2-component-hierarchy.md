## Implementation Notes

### Component Organization
- Follow atomic design strictly for consistency
- Keep atoms in `ui/` folder (shadcn base)
- Molecules and organisms in feature folders
- Co-locate related components

### Naming Conventions
- Components: PascalCase (`WorkflowCard.tsx`)
- Hooks: camelCase with `use` prefix (`useWorkflow.ts`)
- Utils: camelCase (`formatDate.ts`)
- Types: PascalCase (`Workflow`, `ExecutionStatus`)

### Composition Patterns
- Prefer composition over inheritance
- Use render props for flexibility
- Compound components for related UI (e.g., `Tabs`, `TabsList`, `TabsTrigger`)
- Slots pattern for customizable layouts

### Performance Considerations
- Lazy load heavy components (editor panels)
- Virtualize long lists
- Memoize expensive computations
- Code split by route group
