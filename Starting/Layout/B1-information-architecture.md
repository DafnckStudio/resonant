## Implementation Notes

### Routing Framework
- Use Next.js 16 App Router with file-based routing
- Dynamic routes with `[id]` and catch-all `[...slug]`
- Route groups `(auth)`, `(dashboard)` for layout organization
- Parallel routes for modals (`@modal`)

### Navigation State
- Sidebar collapse state persisted in localStorage
- Active route detection via `usePathname()`
- Breadcrumbs generated from route segments

### Access Control
- Route protection via Clerk middleware (`proxy.ts`)
- Role-based UI with conditional rendering
- API routes check permissions server-side

### Search Implementation
- Command palette with Cmdk component
- Debounced search (300ms)
- Results grouped by entity type
- Keyboard navigation support
